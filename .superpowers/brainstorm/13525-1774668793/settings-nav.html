Structure de navigation — Page Paramètres

Comment organiser les 4 sections de settings ? L'option choisie définit l'UX globale de la page.

A — Tabs horizontaux

4 onglets en haut de page. Navigation classique, contenu unique par onglet. Pattern déjà utilisé dans l'app (CRM analytics).

Pour

  • Familier, rapide
  • Cohérent avec l'existant
  • Tout visible d'un coup d'œil

Contre

  • Peu de place sur mobile

📱 CRM

B — Menu latéral gauche

Sidebar de navigation dédiée à droite du sidebar principal. Pattern type "Settings" d'apps SaaS modernes (Linear, Vercel, Notion).

Pour

  • Scalable (ajout futur facile)
  • Look SaaS premium
  • URL par section (/settings/equipe)

Contre

  • Plus de surface à coder
  • Nested layout

🖥 Caisse
📱 CRM

C — Accordéons sur une page

Toutes les sections sur une seule page longue, chacune expansible/réductible. Pas de navigation entre pages.

Pour

  • Tout sur une page, pas de navigation
  • Simple à implémenter

Contre

  • Peu scalable
  • Difficile si sections longues
